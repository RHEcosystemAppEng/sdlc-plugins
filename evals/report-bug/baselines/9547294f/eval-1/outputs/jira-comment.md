# Bug Report Summary

**Advisory upload fails with 500 when filename contains spaces**

A bug report was created with the following sections:

- **Issue Description**: Advisory file uploads with spaces in the filename cause HTTP 500 errors due to a path-parsing failure in the ingestion pipeline. Both the web UI and REST API (`POST /api/v2/advisory`) are affected.
- **Steps to Reproduce**: Upload a file with spaces in its name (e.g., "my advisory 2024.json") via the Trustify web UI or API.
- **Expected Result**: The advisory is uploaded and ingested successfully regardless of filename spaces.
- **Actual Result**: Server returns 500 Internal Server Error with `InvalidPath: path contains unescaped spaces` in the logs.
- **Attachments**: Browser screenshot, server log excerpt, and sample advisory file provided.
- **Root Cause**: The ingestion pipeline passes the original filename to `std::path::Path::new()` without sanitizing whitespace; the path validator rejects spaces.
- **Suggested Fix**: URL-encode or sanitize the filename before path validation, or relax the validator to allow spaces. Fix location: `modules/ingestor/src/service/mod.rs` in `validate_path()`.
