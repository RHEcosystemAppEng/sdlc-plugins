# Root Cause Analysis — ACME-500

## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md` section headings using `line[3:]`, which preserves trailing whitespace from the source file. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent convention-aware task enrichment step performs an exact-match dictionary lookup against the clean convention name, which fails silently because the key contains trailing whitespace. No warning is emitted, so the convention is silently dropped from the generated task's Implementation Notes.

### What is broken

Heading extraction in the plan-feature convention conformance analysis does not strip trailing whitespace from `CONVENTIONS.md` heading lines. The expression `line[3:]` captures all characters after `## `, including any trailing spaces or tabs.

### Where it is broken

- **File**: `plugins/sdlc-workflow/skills/plan-feature/SKILL.md`
- **Code**: `section_name = line[3:]` in the convention conformance analysis loop
- **Downstream failure**: exact-match lookup `if convention_name in discovered_conventions` in the convention-aware task enrichment step (same file)

### Why it is broken

The `line[3:]` string slicing operation extracts a substring starting at index 3 to the end of the line. It does not apply `.strip()` or `.rstrip()` to normalize the result. While many `CONVENTIONS.md` files may not have trailing whitespace, it is a common occurrence in hand-edited markdown files, and the code does not defend against it.

### How to verify the fix

A reproducer test should:

1. Create a `CONVENTIONS.md` fixture with trailing whitespace on at least one heading (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the plan-feature convention extraction logic against this fixture.

4. Assert that the convention-aware task enrichment includes the convention in the generated task's Implementation Notes — specifically that the output contains `Per CONVENTIONS.md §Migration Patterns:`.

## Suggested Approach

Apply `.strip()` to the heading extraction: change `section_name = line[3:]` to `section_name = line[3:].strip()`. This normalizes the section name by removing both leading and trailing whitespace, making the exact-match lookup resilient to whitespace variations in the source file.
