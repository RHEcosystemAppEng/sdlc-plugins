## Root Cause

The plan-feature skill's convention heading extraction uses `line[3:]` to extract the section name after `## `, but does not strip trailing whitespace from the result. When a `CONVENTIONS.md` heading has trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` (with trailing spaces). The downstream convention matching uses exact string comparison (`convention_name in discovered_conventions`), which fails because the lookup key `"Migration Patterns"` does not match the stored key `"Migration Patterns  "`. The convention is silently dropped with no warning.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention heading extraction logic (`line[3:]` without `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment (exact-match lookup against unstripped keys)

## Suggested Approach

1. Add `.strip()` to the heading extraction: change `section_name = line[3:]` to `section_name = line[3:].strip()` so that trailing whitespace on convention headings is normalized.
2. Optionally, also `.strip()` the lookup key for defense-in-depth.
3. Consider adding a warning log when a convention name referenced during task enrichment is not found in the parsed conventions dictionary, to prevent silent failures in the future.

## Reproducer Strategy

Create a test fixture `CONVENTIONS.md` with trailing whitespace on a heading line (e.g., `## Migration Patterns  ` with trailing spaces). Run the convention extraction and matching logic against this fixture. Assert that the convention is still matched and included in the generated task's Implementation Notes despite the trailing whitespace. The test should fail before the fix (convention silently dropped) and pass after the fix (convention correctly matched).
