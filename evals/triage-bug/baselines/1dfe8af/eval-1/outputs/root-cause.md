## Root Cause

The plan-feature skill's convention heading parser extracts section names from `CONVENTIONS.md` using `line[3:]`, which slices the heading text after the `## ` prefix but does not strip trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted key becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent exact-match lookup in the task enrichment step (`if convention_name in discovered_conventions`) fails because the clean convention name does not match the whitespace-padded dictionary key. The convention is silently dropped with no warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention heading extraction logic (`line[3:]` without `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task enrichment logic (exact-match lookup against unstripped keys)

## Suggested Approach

Apply `.strip()` to the extracted heading text so that trailing (and leading) whitespace is normalized before storing the section name as a dictionary key. Change `section_name = line[3:]` to `section_name = line[3:].strip()`. This ensures that headings with trailing whitespace match correctly during the enrichment lookup.

Optionally, add a warning log when a known convention name is not found in the discovered conventions, to prevent silent failures for other edge cases in the future.

## Reproducer Strategy

Create a test fixture `CONVENTIONS.md` with trailing whitespace on a heading line (e.g., `## Migration Patterns  ` with two trailing spaces). Run the convention heading parser against this fixture and assert that the extracted section name equals `"Migration Patterns"` (without trailing whitespace). Then verify that the task enrichment step includes the convention reference in the Implementation Notes when the heading has trailing whitespace.
