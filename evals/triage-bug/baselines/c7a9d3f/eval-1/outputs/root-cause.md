## Root Cause

The plan-feature skill's convention heading extraction uses `line[3:]` to extract section names from `CONVENTIONS.md`, but does not strip trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent convention-to-task matching performs an exact dictionary lookup, which fails silently because the whitespace-padded key does not match the clean convention name. No warning is logged, so the convention is silently dropped from the generated task description.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — Convention heading extraction logic (`line[3:]` without `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — Convention-aware task enrichment (exact-match dictionary lookup against whitespace-padded keys)

## Suggested Approach

Apply `.strip()` to the extracted heading text so that trailing (and leading) whitespace is normalized before storing the convention name in the dictionary. Change `section_name = line[3:]` to `section_name = line[3:].strip()`. This ensures convention names are clean regardless of whitespace in the source file.

## Reproducer Strategy

Create a test with a `CONVENTIONS.md` fixture that includes trailing whitespace on a heading line (e.g., `## Migration Patterns  ` with trailing spaces). Run the convention extraction logic and verify that:

1. The extracted section name matches the clean heading text (`"Migration Patterns"`, not `"Migration Patterns  "`)
2. The convention-to-task matching successfully includes the convention in the generated task's Implementation Notes
3. The generated output contains the expected `Per CONVENTIONS.md §Migration Patterns: ...` reference
