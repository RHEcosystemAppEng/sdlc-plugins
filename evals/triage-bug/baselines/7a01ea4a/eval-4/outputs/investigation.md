### Investigation Findings

Based on analysis of the repository context:

1. **Endpoint handler**: The `/api/v2/advisories` endpoint has two code paths -- one for unfiltered queries and one for date-range filtered queries. The unfiltered path correctly invokes the pagination header builder, but the filtered path constructs a different query that does not pass the total count through to the response builder.

2. **Pagination header logic**: The pagination headers (`X-Total-Count`, `Link`) are generated by a shared utility that requires the total count of matching records. The filtered query path likely performs the filter but does not execute the count query needed to populate these headers.

3. **Root pattern**: Non-filtered requests follow a path that issues both a data query and a count query. Filtered requests appear to skip the count query or fail to pass the count result to the pagination header builder.
