## Root Cause

The plan-feature skill's convention conformance analysis extracts CONVENTIONS.md section headings
using `line[3:]`, which captures everything after `## ` including any trailing whitespace. When a
heading line has trailing spaces (e.g., `## Migration Patterns  `), the extracted section name
becomes `"Migration Patterns  "` (with trailing spaces). The downstream convention-aware task
enrichment performs an exact-match dictionary lookup using the clean convention name
`"Migration Patterns"`, which does not match the whitespace-padded key. The convention is
silently dropped with no warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention heading extraction logic
  (`section_name = line[3:]` without `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment
  (exact-match lookup against whitespace-padded keys)

## Suggested Approach

1. Apply `.strip()` to the extracted section name in the heading extraction logic:
   ```python
   section_name = line[3:].strip()
   ```
2. Consider also adding a warning log when a convention name lookup fails, to prevent silent
   drops in the future.

## Reproducer Strategy

Create a test fixture CONVENTIONS.md with trailing whitespace on a heading line (e.g.,
`## Migration Patterns  ` with trailing spaces). Run the plan-feature convention conformance
analysis against it and assert that the convention is successfully matched and included in
the generated task's Implementation Notes. The test should:

1. **Fail before fix**: The convention should be silently dropped due to whitespace mismatch.
2. **Pass after fix**: The convention should be matched and included after applying `.strip()`.
