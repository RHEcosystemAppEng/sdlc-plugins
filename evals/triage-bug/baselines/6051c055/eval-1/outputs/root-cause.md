## Root Cause

The plan-feature skill's convention lookup extracts CONVENTIONS.md section headings using `line[3:]`, which preserves trailing whitespace from the heading line. When a heading like `## Migration Patterns  ` (with trailing spaces) is parsed, the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The downstream convention-aware task enrichment performs an exact-match lookup (`if convention_name in discovered_conventions`), which fails because the clean key `"Migration Patterns"` does not match the whitespace-polluted key `"Migration Patterns  "`. The convention is silently dropped with no warning.

## Affected Files

- **`plugins/sdlc-workflow/skills/plan-feature/SKILL.md`** -- Convention conformance analysis section
  - Heading extraction: `section_name = line[3:]` (missing `.strip()`)
  - Convention-aware task enrichment: exact-match lookup against whitespace-polluted keys

## Suggested Approach

Add a `.strip()` call to the heading extraction line so that trailing whitespace is removed before storing the section name:

```python
section_name = line[3:].strip()
```

This ensures that headings with trailing whitespace are normalized to their clean form before being used as dictionary keys. No other changes should be needed -- the downstream exact-match comparison will work correctly once the keys are clean.

2. **Pass after the fix**: the `.strip()` call normalizes the name to `"Migration Patterns"`
