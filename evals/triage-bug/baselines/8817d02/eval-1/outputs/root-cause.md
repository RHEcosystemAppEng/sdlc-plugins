## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md` headings using `line[3:]`, which does not strip trailing whitespace. When a heading line has trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` (with trailing spaces). The subsequent convention lookup uses an exact match against `"Migration Patterns"` (without trailing spaces), which fails silently. The convention is dropped from the generated task's Implementation Notes with no warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention conformance analysis section
  - **Heading extraction**: `section_name = line[3:]` does not call `.strip()` on the extracted heading text
  - **Convention lookup**: `if convention_name in discovered_conventions` performs exact-match comparison, which fails when the key has trailing whitespace

## Suggested Approach

Add `.strip()` to the heading extraction line so that trailing whitespace is removed before the section name is stored in the conventions dictionary:

```python
section_name = line[3:].strip()
```

This ensures that headings with trailing whitespace are normalized before being used as dictionary keys, so the exact-match convention lookup succeeds regardless of trailing whitespace in the source file.

4. Before the fix, this test should fail (convention is silently dropped); after the fix, it should pass
