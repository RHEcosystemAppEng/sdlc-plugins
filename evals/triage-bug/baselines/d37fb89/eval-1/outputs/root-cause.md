## Root Cause

The plan-feature skill's convention lookup extracts heading names from `CONVENTIONS.md` using `line[3:]`, which preserves trailing whitespace from the heading line. When a heading such as `## Migration Patterns  ` (with trailing spaces) is parsed, the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent convention-matching step performs an exact string comparison against the expected name (without trailing spaces), causing the match to fail silently. No warning or error is emitted, so the convention is simply omitted from the generated task.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention conformance analysis section: the heading extraction at `line[3:]` does not call `.strip()` to remove trailing whitespace
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task enrichment section: the exact-match lookup `convention_name in discovered_conventions` fails when keys have trailing whitespace

## Suggested Approach

Add `.strip()` to the heading extraction so that `section_name = line[3:].strip()` normalizes the heading text regardless of trailing whitespace. This ensures the convention dictionary keys are clean and match the expected convention names during the enrichment step.

## Reproducer Strategy

Create a test fixture `CONVENTIONS.md` with trailing whitespace on at least one heading (e.g., `## Migration Patterns  ` with trailing spaces). Run the plan-feature convention lookup against this fixture and assert that:
1. The convention is correctly extracted and keyed as `"Migration Patterns"` (without trailing spaces)
2. The generated task's Implementation Notes include the expected convention reference (`Per CONVENTIONS.md §Migration Patterns: ...`)
