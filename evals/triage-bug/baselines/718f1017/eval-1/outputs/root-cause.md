# Step 4: Root Cause Analysis

## Root Cause

The heading extraction in plan-feature's convention lookup uses `line[3:]` to extract the section name after the `## ` prefix. This preserves any trailing whitespace present on the heading line. When `CONVENTIONS.md` contains a heading like `## Migration Patterns  ` (with trailing spaces), the extracted key becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent exact-match lookup (`convention_name in discovered_conventions`) fails silently, causing the convention to be omitted from generated task descriptions without any warning.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` (heading extraction: `section_name = line[3:]`)

## Suggested Approach

Apply `.strip()` to `line[3:]` so that trailing whitespace on heading lines is removed before storing the section name in the conventions dictionary:

```python
section_name = line[3:].strip()
```

This ensures heading extraction is robust against trailing whitespace in `CONVENTIONS.md` files, regardless of editor or platform formatting differences.

## Reproducer Strategy

1. Create a `CONVENTIONS.md` fixture with trailing whitespace on a heading (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the convention lookup logic against this fixture.
3. Assert that the convention reference (e.g., `Per CONVENTIONS.md §Migration Patterns:`) is included in the generated task's Implementation Notes.
