# Root Cause Analysis — ACME-500

## Comment posted on Bug ACME-500

### Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md` heading text using `line[3:]`, which preserves trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent exact-match lookup against clean convention names fails silently — no warning is logged and the convention is omitted from the generated task's Implementation Notes.

### Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention conformance analysis section (heading extraction: `section_name = line[3:]`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task enrichment section (name matching: `convention_name in discovered_conventions`)

### Suggested Approach

Apply `.strip()` to the extracted heading text at the point of extraction:
```
section_name = line[3:].strip()
```
This normalizes the section name regardless of trailing whitespace in the source file, ensuring exact-match lookups succeed consistently.

### Reproducer Strategy

1. Create a `CONVENTIONS.md` fixture file with trailing whitespace on at least one heading (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the plan-feature skill against a feature that requires the convention.
3. Assert that the generated task's Implementation Notes include the convention reference (e.g., `Per CONVENTIONS.md §Migration Patterns: ...`).
4. The reproducer should fail before the fix (convention silently dropped) and pass after the fix (convention correctly included).
