## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md` headings using `line[3:]`, which takes the substring after `## ` but does **not** strip trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` (with trailing spaces). The subsequent exact-match lookup against the expected convention name `"Migration Patterns"` fails silently, causing the convention to be omitted from the generated task description with no warning.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention conformance analysis section, heading extraction logic (`section_name = line[3:]`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task enrichment section, name matching logic (`if convention_name in discovered_conventions`)

## Suggested Approach

Strip trailing whitespace from the extracted heading text by applying `.strip()` (or `.rstrip()`) to `line[3:]` during extraction. This ensures that convention names are normalized regardless of trailing whitespace in the source file. Additionally, consider logging a warning when a convention heading is found but not matched, to prevent silent failures in the future.

## Reproducer Strategy

Create a test with a `CONVENTIONS.md` fixture that includes trailing whitespace on a heading line (e.g., `## Migration Patterns  ` with two trailing spaces). Run the plan-feature convention lookup against this fixture and assert that:

1. The convention **is** matched despite the trailing whitespace (expected behavior after fix)
2. The generated task's Implementation Notes include the convention reference (e.g., `Per CONVENTIONS.md "Migration Patterns": ...`)

Before the fix, this test should fail (convention is silently dropped). After the fix, it should pass.
