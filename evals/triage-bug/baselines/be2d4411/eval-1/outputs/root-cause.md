## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md` section headings using `line[3:]`, which preserves any trailing whitespace present on the heading line. When a heading is written as `## Migration Patterns  ` (with trailing spaces), the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The downstream convention-aware task enrichment step performs an exact-match lookup against the clean name `"Migration Patterns"`, which fails because the dictionary key contains trailing whitespace. The convention is silently dropped with no warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention conformance analysis section (heading extraction loop: `section_name = line[3:]`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention-aware task enrichment section (exact-match lookup: `convention_name in discovered_conventions`)

## Suggested Approach

1. Apply `.strip()` (or `.rstrip()`) to the extracted section name in the heading extraction loop so that trailing whitespace is normalized before storing the key in the conventions dictionary.
2. Consider adding a warning log when a convention section name contains leading or trailing whitespace, to surface similar issues in the future.

## Reproducer Strategy

Create a reproducer test that:
1. Uses a `CONVENTIONS.md` fixture with trailing whitespace on at least one heading (e.g., `## Migration Patterns  `).
2. Runs the convention conformance analysis against a feature that references the whitespace-affected convention.
3. Asserts that the generated task's Implementation Notes include the convention reference (e.g., `Per CONVENTIONS.md §Migration Patterns: ...`).
4. The test should fail before the fix (convention silently dropped) and pass after (convention correctly included).
