## Root Cause

The plan-feature skill's convention heading parser extracts section names from CONVENTIONS.md
using `line[3:]` without stripping trailing whitespace. When a heading line contains trailing
spaces (e.g., `## Migration Patterns  `), the extracted section name becomes
`"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent convention lookup
in the task enrichment step uses exact string matching (`convention_name in discovered_conventions`),
which fails because the query key has no trailing whitespace while the stored key does. The
convention is silently dropped with no warning or error logged.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention heading extraction logic
  (`section_name = line[3:]` does not call `.strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment
  (exact-match comparison against un-stripped keys)

## Suggested Approach

Apply `.strip()` to the extracted heading text so that trailing (and leading) whitespace is
normalized before storing the section name in the conventions dictionary:

```python
section_name = line[3:].strip()
```

This ensures that headings with trailing whitespace are matched correctly during the task
enrichment lookup. Additionally, consider logging a warning when a heading's raw text differs
from its stripped form, to surface formatting issues in CONVENTIONS.md files.

## Reproducer Strategy

1. Create a CONVENTIONS.md fixture with trailing whitespace on a heading line
   (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the plan-feature convention conformance analysis against a feature that should
   match that convention section.
3. Assert that the generated task's Implementation Notes include the convention reference
   (e.g., `Per CONVENTIONS.md Migration Patterns: ...`).
4. Before the fix, this assertion should fail (convention silently dropped). After the fix,
   it should pass.
