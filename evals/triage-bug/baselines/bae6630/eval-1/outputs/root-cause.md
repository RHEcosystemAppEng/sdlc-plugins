## Root Cause

The plan-feature skill's convention conformance analysis extracts CONVENTIONS.md section headings using `line[3:]`, which preserves any trailing whitespace from the heading line. When a heading like `## Migration Patterns  ` (with trailing spaces) is parsed, the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent convention lookup uses an exact-match comparison (`convention_name in discovered_conventions`), which fails because the stored key includes trailing whitespace that the lookup key does not. The convention is silently dropped with no warning logged.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention conformance analysis section: heading extraction at `line[3:]` does not strip trailing whitespace
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment section: exact-match lookup `convention_name in discovered_conventions` fails on whitespace-mismatched keys

## Suggested Approach

1. Apply `.strip()` to the heading extraction: change `line[3:]` to `line[3:].strip()` so that trailing whitespace on heading lines is normalized before storing in the conventions dictionary.
2. Add a warning log when a convention name referenced during task enrichment is not found in the discovered conventions dictionary, to prevent silent failures in the future.

## Reproducer Strategy

1. Create or extend the eval fixture at `evals/plan-feature/files/conventions-mock.md` to include a convention section with trailing whitespace on the heading (e.g., `## Migration Patterns  `).
2. Run the plan-feature convention conformance analysis against this fixture.
3. Assert that the generated task's Implementation Notes include the convention reference (e.g., `Per CONVENTIONS.md Migration Patterns: ...`).
4. Before the fix, this assertion should fail (convention is silently dropped). After the fix, it should pass.
