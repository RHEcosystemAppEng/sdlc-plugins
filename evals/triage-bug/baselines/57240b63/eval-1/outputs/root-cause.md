## Root Cause

The plan-feature skill's convention conformance analysis extracts `CONVENTIONS.md` section headings using `line[3:]`, which captures all text after the `## ` prefix without stripping trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted key becomes `"Migration Patterns  "` (with trailing spaces). The subsequent convention-aware task enrichment performs an exact-match lookup using clean convention names (e.g., `"Migration Patterns"`), which fails against the un-stripped key. The convention is silently dropped with no warning or error logged.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — Convention conformance analysis section: heading extraction logic (`section_name = line[3:]`) does not call `.strip()` on the extracted heading name.
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — Convention-aware task enrichment section: exact-match lookup (`if convention_name in discovered_conventions`) assumes keys are clean, which fails when they contain trailing whitespace.

## Suggested Approach

1. Apply `.strip()` to the heading extraction: change `section_name = line[3:]` to `section_name = line[3:].strip()` to normalize heading names at extraction time.
2. Optionally, add a defensive `.strip()` on the lookup side as well for belt-and-suspenders safety.
3. Consider logging a warning when a `CONVENTIONS.md` heading contains leading or trailing whitespace, to surface formatting issues to users.

## Reproducer Strategy

Create a test fixture `CONVENTIONS.md` with trailing whitespace on at least one heading line (e.g., `## Migration Patterns  ` with trailing spaces). Run the plan-feature convention lookup against this fixture and assert that the convention is correctly matched despite the trailing whitespace. The reproducer test should:

1. **Before fix**: Assert that the convention IS NOT matched (confirming the bug exists).
2. **After fix**: Assert that the convention IS matched and appears in the generated task's Implementation Notes as `Per CONVENTIONS.md §Migration Patterns: ...`.
