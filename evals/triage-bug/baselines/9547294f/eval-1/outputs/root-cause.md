## Root Cause

The plan-feature skill's convention heading extraction uses `line[3:]` to extract section names from `CONVENTIONS.md` headings. This preserves any trailing whitespace present on the heading line. When the convention-aware task enrichment step later performs an exact-match lookup against the extracted section names, headings with trailing whitespace fail to match their expected (trimmed) names. The convention is silently dropped from the generated task description with no warning or error.

The core defect is the missing `.strip()` (or equivalent whitespace trimming) call on the extracted heading text at the point of extraction.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention heading extraction logic (`section_name = line[3:]` should be `section_name = line[3:].strip()`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment (consumer of the incorrectly-keyed dictionary; no change needed here if extraction is fixed)

## Suggested Approach

1. Add `.strip()` to the heading extraction: change `section_name = line[3:]` to `section_name = line[3:].strip()` so that trailing whitespace on heading lines is normalized before storing in the conventions dictionary.
2. Optionally, add a defensive `.strip()` on the lookup side as well for resilience.
3. Add a reproducer eval fixture with trailing whitespace on convention headings to prevent regression.

## Reproducer Strategy

Create a test `CONVENTIONS.md` fixture with trailing whitespace on at least one heading (e.g., `## Migration Patterns  ` with trailing spaces). Run the plan-feature convention conformance analysis against this fixture and assert that:
- The convention IS included in the generated task's Implementation Notes (expected behavior)
- Before the fix: the convention is missing (confirming the bug)
- After the fix: the convention appears correctly (confirming the fix)
