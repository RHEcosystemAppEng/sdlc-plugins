## Root Cause

The plan-feature skill's convention conformance analysis extracts CONVENTIONS.md section headings using `line[3:]`, which captures all characters after `## ` including any trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted key becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The downstream convention matching uses exact string comparison (`if convention_name in discovered_conventions`), which fails because the lookup key does not include the trailing whitespace. The convention is silently skipped with no warning.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention conformance analysis section (heading extraction logic using `line[3:]`)
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment section (exact-match convention name lookup)

## Suggested Approach

1. Strip trailing whitespace from the extracted heading text after `line[3:]` (e.g., `line[3:].strip()` or `.rstrip()`).
2. Optionally, also strip trailing whitespace from the lookup key to ensure both sides of the comparison are normalized.
3. Consider adding a warning/log message when a convention heading is found but does not match any expected convention name, to prevent silent failures in the future.

## Reproducer Strategy

Create a `CONVENTIONS.md` fixture file with trailing whitespace on one or more section headings. Run the plan-feature convention loading logic against this fixture and assert that:
- The convention section is found and included in the `discovered_conventions` dictionary with a clean (stripped) key.
- The convention-aware task enrichment includes the convention reference in the generated task's Implementation Notes.
- Before the fix: the test should fail (convention is silently dropped).
- After the fix: the test should pass (convention is correctly matched and included).
