# Step 4 -- Root Cause Analysis

## Root Cause

The convention heading extraction logic in `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` uses `line[3:]` to extract the heading text after the `## ` prefix, but does not strip trailing whitespace. When a `CONVENTIONS.md` file contains headings with trailing spaces (e.g., `## Migration Patterns  `), the extracted section name retains those spaces (`"Migration Patterns  "` instead of `"Migration Patterns"`).

Downstream, the convention-aware task enrichment performs an exact-match lookup (`convention_name in discovered_conventions`), which fails because the canonical convention name (without trailing whitespace) does not match the extracted name (with trailing whitespace). This causes convention-driven settings -- including user preference persistence such as dark mode -- to silently fail to apply.

## Affected Files

| File | Symbol | Defect |
|------|--------|--------|
| `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` | Heading extraction: `section_name = line[3:]` | Missing `.strip()` call on extracted heading text |
| `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` | Task enrichment: `convention_name in discovered_conventions` | Exact-match fails when keys have trailing whitespace |

## Suggested Approach

Apply `.strip()` to the heading text after extraction to normalize whitespace:

```python
section_name = line[3:].strip()
```

This ensures that convention section names are always trimmed, regardless of trailing whitespace in the source `CONVENTIONS.md` file. The downstream exact-match lookup will then work correctly.
