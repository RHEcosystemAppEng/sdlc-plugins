## Root Cause

The plan-feature skill's convention conformance analysis extracts CONVENTIONS.md headings using `line[3:]`, which slices everything after `## ` but does **not** strip trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` (with trailing spaces). The downstream task enrichment step performs an exact-match comparison (`if convention_name in discovered_conventions`), which expects `"Migration Patterns"` (without trailing spaces). The match fails, and the convention is silently dropped with no warning or error.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention heading extraction logic
  - Defective expression: `section_name = line[3:]` (missing `.strip()`)
  - Downstream match: `if convention_name in discovered_conventions` (exact match against un-stripped keys)

## Suggested Approach

1. Apply `.strip()` to the extracted heading text: change `line[3:]` to `line[3:].strip()` so trailing whitespace is removed before storing the section name as a dictionary key.
2. Consider also adding a warning log when a convention heading is found during extraction but fails to match during task enrichment, to prevent future silent failures.

## Reproducer Strategy

A reproducer test should:
1. Create a CONVENTIONS.md fixture with trailing whitespace on a heading line (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the convention extraction logic against this fixture.
3. Assert that the extracted section name is `"Migration Patterns"` (stripped), not `"Migration Patterns  "` (with trailing spaces).
4. Verify that convention-aware task enrichment correctly matches and includes the convention in the generated task's Implementation Notes.
