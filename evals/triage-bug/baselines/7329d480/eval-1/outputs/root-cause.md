## Root Cause

The plan-feature skill's convention lookup extracts CONVENTIONS.md heading text using `line[3:]`, which captures everything after `## ` without stripping trailing whitespace. When a heading line contains trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes `"Migration Patterns  "` instead of `"Migration Patterns"`. The subsequent exact-match comparison against the expected convention name fails silently, causing the convention to be omitted from the generated task description with no warning.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` — convention heading extraction uses `line[3:]` without `.strip()`, and convention-aware task enrichment performs exact match against keys with potential trailing whitespace.

## Suggested Approach

1. Apply `.strip()` to the heading extraction: change `section_name = line[3:]` to `section_name = line[3:].strip()` to normalize heading text by removing leading and trailing whitespace.
2. Consider adding a warning log when a convention is expected but not found in the discovered conventions, to prevent silent failures in the future.

## Reproducer Strategy

Create a test that:
1. Provides a CONVENTIONS.md fixture with trailing whitespace on a heading line (e.g., `## Migration Patterns  ` with trailing spaces).
2. Runs the convention extraction logic against this fixture.
3. Asserts that the convention is discovered and matched correctly despite the trailing whitespace.
4. Verifies that the generated task's Implementation Notes include the expected convention reference (`Per CONVENTIONS.md §Migration Patterns: ...`).
