## Root Cause

The plan-feature skill's convention extraction logic uses `line[3:]` to extract heading text
from `CONVENTIONS.md`, but does not strip trailing whitespace. When a heading line contains
trailing spaces (e.g., `## Migration Patterns  `), the extracted section name becomes
`"Migration Patterns  "` instead of `"Migration Patterns"`. The downstream task enrichment
step performs an exact-match lookup against the expected convention name (without trailing
spaces), which fails silently -- no warning is emitted and the convention is omitted from the
generated task.

## Affected Files

- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention extraction loop
  (`section_name = line[3:]`) lacks `.strip()` call
- `plugins/sdlc-workflow/skills/plan-feature/SKILL.md` -- convention-aware task enrichment
  (`convention_name in discovered_conventions`) fails due to whitespace mismatch in dictionary key

## Suggested Approach

1. Add `.strip()` to the heading extraction: change `line[3:]` to `line[3:].strip()` so that
   trailing whitespace is removed from section names before they are stored in the conventions
   dictionary.
2. Optionally, add a diagnostic warning when a heading line's raw text differs from its stripped
   form, to aid debugging of similar issues in the future.

## Reproducer Strategy

1. Create a test fixture `CONVENTIONS.md` with trailing whitespace on at least one heading
   (e.g., `## Migration Patterns  ` with two trailing spaces).
2. Run the convention extraction logic against this fixture.
3. Assert that the extracted section name equals `"Migration Patterns"` (without trailing spaces).
4. Assert that the convention-aware task enrichment correctly includes the convention in the
   generated task's Implementation Notes.
