# Step 3 — Affects Versions Correction: TC-8001

## Current vs Corrected Affects Versions

| | Affects Versions |
|---|---|
| **PSIRT-assigned (current)** | RHTPA 2.0.0 |
| **Corrected (from lock file evidence)** | RHTPA 2.2.0, RHTPA 2.2.1, RHTPA 2.2.2 |

## Rationale

PSIRT assigned **RHTPA 2.0.0** as the Affects Version, but there is no 2.0.x version stream configured in the project. The issue is scoped to the **2.2.x** stream (per the summary suffix `[rhtpa-2.2]`).

Lock file analysis shows that quinn-proto was upgraded to the fixed version (0.11.14) starting in build 0.4.11 (version 2.2.3). Therefore:

- **RHTPA 2.2.0** — ships quinn-proto 0.11.9 — AFFECTED
- **RHTPA 2.2.1** — ships quinn-proto 0.11.12 — AFFECTED
- **RHTPA 2.2.2** — retag of 2.2.1, ships quinn-proto 0.11.12 — AFFECTED
- **RHTPA 2.2.3** — ships quinn-proto 0.11.14 — NOT AFFECTED (fixed)
- **RHTPA 2.2.4** — ships quinn-proto 0.11.14 — NOT AFFECTED (fixed)

## Proposed Jira Update

Remove RHTPA 2.0.0 and set Affects Versions to: **RHTPA 2.2.0, RHTPA 2.2.1, RHTPA 2.2.2**.
