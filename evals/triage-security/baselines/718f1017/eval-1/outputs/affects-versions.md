## 3.2 — Affects Versions Comparison and Correction

### Stream Scope

This issue has stream suffix `[rhtpa-2.2]` -> scoped to stream **2.2.x**.
Only versions belonging to the 2.2.x stream are included in the Affects Versions correction.

### Version Impact (2.2.x stream only)

| Version | quinn-proto | Affected? |
|---------|-------------|-----------|
| 2.2.0 | 0.11.9 | **YES** |
| 2.2.1 | 0.11.12 | **YES** |
| 2.2.2 | — (retag of 2.2.1) | **YES** |
| 2.2.3 | 0.11.14 | NO (ships fixed version) |
| 2.2.4 | 0.11.14 | NO (ships fixed version) |

### Correction

The PSIRT-assigned Affects Versions is **wrong**:

- **Current**: `[RHTPA 2.0.0]`
- **Problem**: RHTPA 2.0.0 does not correspond to any configured Version Stream (no 2.0.x stream exists). This is an incorrect PSIRT assignment.
- **Proposed**: `[RHTPA 2.2.0, RHTPA 2.2.1, RHTPA 2.2.2]`

These are the 2.2.x stream versions that ship quinn-proto < 0.11.14 (the vulnerable range), based on lock file analysis at pinned commits from the supportability matrix.

Versions 2.2.3 and 2.2.4 are excluded because they ship quinn-proto 0.11.14 (the fixed version).
