TC-8003 (CVE-2026-31812 quinn-proto - Panic on large stream counts [rhtpa-2.2]) should be **closed as Duplicate** referencing TC-7999.

## Rationale

### 1. Same CVE

Both TC-8003 and TC-7999 track the same vulnerability: **CVE-2026-31812**, a denial-of-service vulnerability in the quinn-proto Rust crate (versions before 0.11.14).

### 2. Same Stream

Both issues carry the stream suffix **[rhtpa-2.2]**, mapping to the **2.2.x** product version stream (Konflux release repo `rhtpa-release.0.4.z`). PSIRT creates one Vulnerability issue per CVE per stream; having two issues for the same CVE and stream is redundant.

### 3. TC-7999 Is Already In Progress

TC-7999 has status **In Progress**, meaning remediation work has already begun. TC-7999 also has broader Affects Versions coverage:

| Version | TC-8003 | TC-7999 |
|---------|---------|---------|
| RHTPA 2.2.0 | Yes | Yes |
| RHTPA 2.2.1 | No | Yes |

TC-7999 covers RHTPA 2.2.0 and RHTPA 2.2.1, while TC-8003 only lists RHTPA 2.2.0. All versions on TC-8003 are already covered by TC-7999.

### 4. No Remediation Tasks Created

Because this is a same-stream duplicate, the triage flow short-circuits at Step 4.1. No remediation tasks are created for TC-8003 -- all remediation is handled through TC-7999. Steps 5, 6, and 7 are not executed.
