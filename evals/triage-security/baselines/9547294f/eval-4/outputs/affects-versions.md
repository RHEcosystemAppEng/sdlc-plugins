Since TC-8004 is an **unscoped** issue (no stream suffix), the Affects Versions should include all actually affected versions across all streams. The version impact table shows that only 2.1.x versions are affected -- 2.2.x versions ship h2 >= 0.4.8 and are not affected.

### Rationale

PSIRT assigned Affects Versions [RHTPA 2.1.0, RHTPA 2.2.0] based on scan-time assessment. Lock file analysis at pinned commits from security-matrix.md shows:

1. **RHTPA 2.1.0**: h2 0.4.5 at tag `v0.3.8` -- **affected** (correct)
2. **RHTPA 2.1.1**: h2 0.4.5 at tag `v0.3.12` -- **affected** (was missing, should be added)
3. **RHTPA 2.2.0**: h2 0.4.8 at tag `v0.4.5` -- **not affected** (should be removed)

The correction scopes Affects Versions to only the versions that actually ship the vulnerable dependency (h2 < 0.4.8).
