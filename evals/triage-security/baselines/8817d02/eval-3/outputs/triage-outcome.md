# Triage Outcome for TC-8003

## Decision: Close as Duplicate

TC-8003 should be **closed as Duplicate** of TC-7999.

## Rationale

1. **Same CVE**: Both TC-8003 and TC-7999 track CVE-2026-31812 (quinn-proto denial of service via excessive stream counts).

2. **Same stream**: Both issues carry the stream suffix `[rhtpa-2.2]`, placing them in the 2.2.x version stream. PSIRT creates one Vulnerability issue per stream per CVE; having two for the same stream is a duplication error.

3. **Sibling is already active**: TC-7999 has status **In Progress**, meaning remediation work has already started. TC-8003 is still in **New** status.

4. **Sibling has better Affects Versions coverage**: TC-7999 already has `[RHTPA 2.2.0, RHTPA 2.2.1]` which covers all affected 2.2.x versions. TC-8003 only has `[RHTPA 2.2.0]`, which is incomplete. Closing TC-8003 avoids redundant version tracking.

5. **Version impact confirms no gap**: The lock file analysis shows:
   - RHTPA 2.2.0 (v0.4.5): quinn-proto 0.11.9 -- AFFECTED
   - RHTPA 2.2.1 (v0.4.8): quinn-proto 0.11.12 -- AFFECTED
   - RHTPA 2.2.2 (v0.4.9): retag of 2.2.1 -- AFFECTED
   - RHTPA 2.2.3 (v0.4.11): quinn-proto 0.11.14 -- NOT AFFECTED (fixed version)
   - RHTPA 2.2.4 (v0.4.12): quinn-proto 0.11.14 -- NOT AFFECTED (fixed version)

   TC-7999's Affects Versions already capture the affected versions. No additional versions need tracking.
