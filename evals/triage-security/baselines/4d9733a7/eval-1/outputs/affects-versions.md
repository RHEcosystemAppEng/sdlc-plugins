# Step 3 -- Affects Versions Correction

## 3.1 -- Dynamic Version Discovery

**PROPOSED ACTION**: Discover available Jira versions dynamically using `getJiraIssueTypeMetaWithFields` for the Vulnerability issue type (ID: 10024) in project TC.

The Jira version names are referenced from the supportability matrix rather than hardcoded Jira version IDs (Important Rule 6). The version IDs would be discovered at runtime via the API.

## 3.2 -- Compare and Correct Affects Versions

**Scope**: This issue is scoped to the **2.2.x** stream (suffix [rhtpa-2.2]). Only versions belonging to the 2.2.x stream are included in the correction.

### Version Impact (2.2.x stream only)

| Version | quinn-proto | Affected? |
|---------|-------------|-----------|
| RHTPA 2.2.0 | 0.11.9 | YES |
| RHTPA 2.2.1 | 0.11.12 | YES |
| RHTPA 2.2.2 | 0.11.12 | YES (retag of 2.2.1) |
| RHTPA 2.2.3 | 0.11.14 | NO |
| RHTPA 2.2.4 | 0.11.14 | NO |

### Proposed Correction

```
Current: [RHTPA 2.0.0]
Proposed: [RHTPA 2.2.0, RHTPA 2.2.1, RHTPA 2.2.2]
```

**Rationale**: PSIRT assigned "RHTPA 2.0.0" which does not correspond to any version in the supportability matrix for the 2.2.x stream. Lock file analysis at pinned commits from security-matrix.md confirms that versions 2.2.0, 2.2.1, and 2.2.2 ship quinn-proto < 0.11.14 (the affected range). Versions 2.2.3 and 2.2.4 ship quinn-proto 0.11.14 (the fixed version) and are not affected.
