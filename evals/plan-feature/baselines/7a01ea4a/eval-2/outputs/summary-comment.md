## Architecture Summary

The plan decomposes the "Improve search experience" feature into three independent backend tasks targeting the trustify-backend search module:

1. **Performance layer** (Task 1): Adds database indexes on searchable columns (sbom.name, advisory.title, package.name) and timestamp columns. Uses a new SeaORM migration. This is a pure DDL change that improves query execution time without altering search behavior.

2. **Relevance layer** (Task 2): Enhances the SearchService to compute relevance scores using PostgreSQL full-text search ranking functions (ts_rank/ts_rank_cd) and orders results by match quality. Changes are isolated to the service layer query builder.

3. **Filtering layer** (Task 3): Extends the GET /api/v2/search endpoint with optional entity_type, created_after, and created_before query parameters. All parameters are optional and backward-compatible. Adds filter validation (HTTP 400 for invalid values) and WHERE clause construction using the shared query helpers.

All three tasks are independent and can merge to main in any order (direct-to-main workflow mode). Each task includes its own integration tests.
