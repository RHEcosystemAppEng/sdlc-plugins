## Architecture Summary

The feature adds a new `GET /api/v2/sbom/{id}/advisory-summary` endpoint to the trustify-backend that returns aggregated severity counts (critical, high, medium, low, total) for a given SBOM. The implementation follows the existing module pattern (model + service + endpoints) within the `modules/fundamental/src/sbom/` module. The aggregation query uses the existing `sbom_advisory` join table with deduplication by advisory ID — no new database tables are required. Responses are cached for 5 minutes using `tower-http` caching middleware, with cache invalidation triggered by the advisory ingestion pipeline when new advisories are linked. A non-MVP `?threshold` query parameter enables alerting integrations to filter counts by severity level.
