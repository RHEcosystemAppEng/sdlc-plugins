## Architecture Summary

The feature adds a server-side advisory severity aggregation endpoint to replace client-side counting. The implementation follows the existing module pattern (model + service + endpoints):

- **Data layer:** New `SeveritySummary` model and `SbomService::get_advisory_summary` method that queries the existing `sbom_advisory` join table, deduplicates by advisory ID, and groups by severity level.
- **HTTP layer:** New Axum endpoint at `GET /api/v2/sbom/{id}/advisory-summary` with `tower-http` 5-minute caching and optional `?threshold` query parameter.
- **Ingestion layer:** Cache invalidation in the advisory ingestion pipeline to ensure fresh counts after new advisories are linked.
- **No new database tables** — uses existing `sbom_advisory` relationship table.
