## Architecture Summary

The license compliance report feature is implemented as a new sub-module within the existing SBOM domain module, following the established model/service/endpoints pattern:

1. **License policy model** (`model/license_policy.rs`) — configurable compliance rules loaded from a JSON file, defining allowed and denied SPDX license identifiers
2. **License report model and service** (`model/license_report.rs`, `service/license_report.rs`) — core business logic that queries packages for an SBOM, walks the transitive dependency tree via `sbom_package` and `package_license` entities, groups packages by license type, and checks each group against the policy

4. **Integration tests** (`tests/api/license_report.rs`) — end-to-end tests against PostgreSQL
5. **Documentation** — new content covering the endpoint contract and policy configuration

No new database tables are introduced. The report aggregates from existing `package_license` and `sbom_package` entities.
