## Architecture Summary

The backend introduces a new `remediation` module following the established `model/service/endpoints` pattern. Two new REST endpoints (`GET /api/v2/remediation/summary` and `GET /api/v2/remediation/by-product`) compute aggregations from existing vulnerability, SBOM, and package entity data without requiring new database tables. The frontend adds a new page at `/remediation` using PatternFly 5 components, React Query hooks for data fetching, and a filterable table with server-side pagination to handle up to 10,000 vulnerabilities.
