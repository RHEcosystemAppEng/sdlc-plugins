## Architecture Summary

The feature adds a new REST API endpoint `GET /api/v2/sbom/{id}/advisory-summary` that returns pre-computed severity counts for advisories linked to a given SBOM. The implementation follows the existing module pattern (model + service + endpoints) within the `sbom` module of the `fundamental` crate:

1. **Model layer**: new `AdvisorySeveritySummary` response struct
2. **Service layer**: aggregation query joining `sbom_advisory` and `advisory` entities with deduplication
3. **Endpoint layer**: Axum handler with 5-minute tower-http cache
4. **Ingestion pipeline**: cache invalidation hook in the advisory correlation step
5. **Non-MVP**: optional `?threshold` query parameter for severity filtering

No new database tables are required — the endpoint uses existing `sbom_advisory` and `advisory` entities.
