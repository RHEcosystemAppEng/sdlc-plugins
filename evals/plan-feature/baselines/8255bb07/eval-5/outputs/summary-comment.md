### Architecture Summary

This feature replaces the `advisory_status` lookup table with a PostgreSQL enum column (`advisory_status_enum`) directly on the `advisory` table. The changes span the full stack:

1. **Database layer**: A single atomic migration creates the enum type, adds the column, backfills from the existing join, drops the FK, and drops the lookup table
2. **Entity layer**: SeaORM entity definitions updated to use `DeriveActiveEnum` for the new enum type; the `advisory_status` entity is removed
3. **Service/API layer**: All advisory queries simplified by removing the `advisory_status` join; status filtering uses the enum column directly
4. **Ingestion layer**: Advisory ingestion writes enum values directly instead of inserting into the lookup table first
5. **Test layer**: Integration tests updated to verify the new schema and maintain API backward compatibility
