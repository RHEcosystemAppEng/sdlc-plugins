## Architecture Summary

The feature adds a side-by-side SBOM comparison capability across two repositories:

**Backend (trustify-backend):** A new comparison service computes structural diffs between two SBOMs on-the-fly from existing SBOM-package and SBOM-advisory data (no new database tables). The service produces six diff categories: added packages, removed packages, version changes, new vulnerabilities, resolved vulnerabilities, and license changes. A new REST endpoint exposes this service at `GET /api/v2/sbom/compare?left={id1}&right={id2}`.

**Frontend (trustify-ui):** A new comparison page at `/sbom/compare` consumes the backend endpoint via a React Query hook. The page follows Figma specifications with PatternFly 5 components: typeahead SBOM selectors, collapsible diff sections with count badges, data tables with sortable columns, critical vulnerability highlighting, empty and loading states, and JSON/CSV export. The SBOM list page is enhanced with checkbox selection and a "Compare selected" button for navigation. URLs encode both SBOM IDs for shareable comparison links.

**Cross-repo dependency ordering:** Backend tasks (1-2) complete before frontend tasks (3-5) to ensure the API endpoint exists when the frontend is implemented.
