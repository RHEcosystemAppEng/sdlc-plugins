## Architecture Summary

The plan addresses three MVP requirements from TC-9002 with independent, incrementally deliverable tasks targeting the `trustify-backend` search module:

1. **Performance** (Task 1): Adds a database migration with GIN indexes for full-text search and refactors `SearchService` to use PostgreSQL `ts_vector`/`ts_query` operators for indexed query execution.
2. **Relevance** (Task 2): Implements `ts_rank()` scoring in the search query and adds an optional `sort` query parameter to the search endpoint, defaulting to relevance-ranked ordering.
3. **Filtering** (Task 3): Adds optional filter query parameters (`entity_type`, `created_after`, `created_before`) to the search endpoint, leveraging the existing shared query builder helpers in `common/src/db/query.rs`.

All changes are additive and backward compatible. The existing `GET /api/v2/search` endpoint contract is preserved; new parameters are optional.
