## Architecture Summary

The license compliance report feature adds a new `GET /api/v2/sbom/{id}/license-report` endpoint to the trustify-backend SBOM module. The implementation follows the established module pattern (`model/ + service/ + endpoints/`):

1. **Model layer** (Task 1): A `LicensePolicy` configuration model in `common/` loads a JSON policy file defining allowed and denied licenses. A `LicenseReport` response model in the SBOM module defines the grouped response structure.
2. **Service layer** (Task 2): A `LicenseReportService` aggregates package license data from existing entities (`sbom_package`, `package_license`), walks the transitive dependency tree, groups packages by license, and flags compliance against the policy. No new database tables are required.
3. **Endpoint layer** (Task 3): A new Axum handler at `GET /api/v2/sbom/{id}/license-report` wires the service to the HTTP API, following existing endpoint patterns.
4. **Testing** (Task 4): Integration tests in `tests/api/` verify the full request-response cycle including compliance flagging, transitive dependency inclusion, and error handling.
5. **Documentation** (Task 5): New content documenting the endpoint and license policy configuration for compliance officers and CI/CD pipeline integrators.
