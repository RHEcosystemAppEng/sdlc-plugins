## Architecture Summary

The feature adds a vulnerability remediation tracking dashboard with a backend-frontend split:

**Backend (trustify-backend):** A new `remediation` module following the existing model/service/endpoints pattern provides two REST endpoints:
- `GET /api/v2/remediation/summary` -- aggregates vulnerability counts by severity (Critical/High/Medium/Low) x status (Open/In Progress/Resolved)
- `GET /api/v2/remediation/by-product` -- returns paginated per-product remediation breakdown

Aggregation queries operate on existing entity tables (`advisory`, `sbom_advisory`, `package`, `sbom_package`) without creating new database tables, meeting the no-migration NFR. Performance target: p95 < 500ms for the summary endpoint.

**Frontend (trustify-ui):** A new RemediationDashboardPage at `/remediation` with three sections:
1. Summary cards showing Open, In Progress, and Resolved totals
2. Progress chart showing 30-day remediation trend
3. Filterable vulnerability table with severity, product, and status filters

Uses PatternFly 5 components, React Query hooks for data fetching, and existing shared components (SeverityBadge, FilterToolbar, EmptyStateCard, LoadingSpinner). Supports up to 10,000 tracked vulnerabilities via pagination.
