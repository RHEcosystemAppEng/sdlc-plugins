## Architecture Summary

The feature adds a side-by-side SBOM comparison capability across two repositories:

1. **Backend (trustify-backend):** A new `GET /api/v2/sbom/compare?left={id1}&right={id2}` endpoint computes a structured diff between two SBOMs on-the-fly using existing package, advisory, and license data (no new database tables). The response includes six diff categories: added packages, removed packages, version changes, new vulnerabilities, resolved vulnerabilities, and license changes. The comparison service logic lives in `SbomService` and reuses existing `PackageService` and `AdvisoryService` for data retrieval.

2. **Frontend (trustify-ui):** A new comparison page at `/sbom/compare` with a header toolbar (two PatternFly Select typeahead dropdowns, Compare button, Export dropdown) and six collapsible PatternFly `ExpandableSection` diff sections with data tables. The SBOM list page gains checkbox selection and a "Compare selected" button. The comparison URL encodes both SBOM IDs as query params for bookmarking and sharing. Critical vulnerabilities receive highlighted styling.

3. **Non-MVP (Task 8):** Client-side export of comparison results as JSON or CSV for compliance documentation.

4. **Documentation (Task 10):** New API reference for the comparison endpoint and user guide for the comparison workflow.
