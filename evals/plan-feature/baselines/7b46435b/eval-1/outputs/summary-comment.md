## Architecture Summary

The feature adds a new GET /api/v2/sbom/{id}/advisory-summary endpoint to the trustify-backend service. The implementation follows the existing module pattern (model + service + endpoints):

1. **Model layer**: new AdvisorySeveritySummary struct with severity count fields
2. **Service layer**: aggregation query joining sbom_advisory and advisory tables, deduplicating by advisory ID, grouping by severity
3. **Endpoint layer**: Axum handler with path parameter extraction, 404 handling for missing SBOMs, and 5-minute tower-http caching
4. **Cache invalidation**: hook in the advisory ingestion pipeline to invalidate cached summaries when new advisories are linked
5. **Threshold filtering** (non-MVP): optional ?threshold=severity query parameter for severity-level filtering
6. **Documentation**: REST API reference updates for the new endpoint

No new database tables are required — the implementation uses the existing sbom_advisory join table and advisory entity.
