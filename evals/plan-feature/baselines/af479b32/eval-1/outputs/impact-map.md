## Feature: TC-9001 — Add advisory severity aggregation endpoint

### trustify-backend

Changes:
- Add `AdvisorySeveritySummary` response model struct to represent aggregated severity counts (`critical`, `high`, `medium`, `low`, `total`)
- Add severity aggregation method to `SbomService` that queries the `sbom_advisory` join table, deduplicates by advisory ID, and groups counts by severity level
- Add `GET /api/v2/sbom/{id}/advisory-summary` endpoint handler with 5-minute `tower-http` cache configuration and 404 response for nonexistent SBOM IDs
- Register the new advisory-summary route in SBOM endpoint module
- Add cache invalidation hook in advisory ingestion pipeline to invalidate cached advisory summaries when new advisories are linked to an SBOM
- Add optional `?threshold` query parameter support to filter severity counts above a specified level (non-MVP)
- Add integration tests for the advisory-summary endpoint covering happy path, 404, deduplication, caching, and threshold filtering
