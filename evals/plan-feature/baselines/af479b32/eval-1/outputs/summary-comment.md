## Architecture Summary

The feature adds a new REST endpoint `GET /api/v2/sbom/{id}/advisory-summary` that returns aggregated advisory severity counts for a given SBOM. The implementation follows the existing module pattern (model + service + endpoints):

1. **Model layer**: new `AdvisorySeveritySummary` struct with severity count fields
2. **Service layer**: aggregation method in `SbomService` that queries the `sbom_advisory` join table, deduplicates by advisory ID, and groups by severity
3. **Endpoint layer**: Axum handler with 5-minute `tower-http` cache and 404 for nonexistent SBOMs
4. **Cache invalidation**: hook in the advisory ingestion pipeline to invalidate cached summaries when new advisories are linked
5. **Non-MVP extension**: optional `?threshold` query parameter for severity filtering
6. **Testing**: integration tests, smoke tests, and performance benchmarks

No new database tables are required -- the feature uses existing `sbom_advisory` and `advisory` entity tables.
