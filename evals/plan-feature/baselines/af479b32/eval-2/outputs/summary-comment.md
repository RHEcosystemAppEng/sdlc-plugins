### Architecture Summary

The plan addresses three MVP requirements through incremental, backward-compatible changes to the trustify-backend search module:

1. **Performance** (Task 1): Add PostgreSQL GIN indexes on `tsvector` columns for SBOM, advisory, and package entities. Update `SearchService` to use full-text search operators instead of LIKE/ILIKE patterns.

2. **Relevance** (Task 2): Compute relevance scores using PostgreSQL `ts_rank` and sort results by relevance by default when a text query is present. Add an optional `sort` parameter for user override.

3. **Filtering** (Task 3): Add optional query parameters (`entity_type`, `date_from`, `date_to`, `severity`) to the search endpoint with AND-combination semantics. Reuse shared filtering helpers from `common/src/db/query.rs`.
