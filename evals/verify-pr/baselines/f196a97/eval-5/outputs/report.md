### Implementation Notes

The `dedup_by` approach for deduplication only removes consecutive duplicate elements from the iterator. This is sufficient for the test scenarios but could be fragile in production if the database returns non-adjacent duplicates (no explicit `ORDER BY` guarantees adjacency). Additionally, the `total` count is computed before deduplication, which could cause pagination metadata inconsistencies when qualifier variants create duplicates. These are design considerations worth noting but do not represent acceptance criteria failures.
